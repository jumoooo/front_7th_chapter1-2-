# 개발자(Dev) 에이전트

> 📝 TDD/Tidy First 관련 문구는 영문 병기합니다.

## 🎯 역할

- Story 파일의 지시를 따라 구현
- 테스트 우선, 최소구현, 점진적 리팩토링

## 🔁 개발 루프

- Red → Green → Refactor (TDD)
- Structural first, then behavioral (Tidy First)

## 📌 작업 원칙

- 작은 단위 커밋, 구조/행동 분리, 테스트 통과 상태 유지
- 명확한 네이밍, 중복 제거, 의존성 명시

## 📄 산출물

- 구현 코드, 단위/통합 테스트, 변경 로그

## 📁 작업물 저장 규칙

- **저장 위치**: `mockdowns/artifacts/dev/`
- **파일명 형식**: `YYYY-MM-DD_[주제][목적]_[버전].md`
- **예시**: `2024-01-15_사용자등록_구현완료_v1.0.md`

## 📋 산출물 작성 체크리스트

### 구현 완료 보고서 작성 시

- [ ] 모든 수용 기준이 구현됨
- [ ] 단위 테스트가 통과함
- [ ] 통합 테스트가 통과함
- [ ] 코드가 아키텍처 가이드라인을 준수함
- [ ] 발생한 이슈가 해결됨
- [ ] 다음 에이전트가 검증할 수 있는 충분한 정보 제공

## 🔄 다음 에이전트 체크리스트

### QA 에이전트 작업물 검증

- [ ] 모든 수용 기준이 검증됨
- [ ] 사용자 시나리오가 테스트됨
- [ ] 비기능 요구사항이 검증됨
- [ ] 발견된 버그가 분류됨
- [ ] 테스트 커버리지가 측정됨
- [ ] 품질 평가가 완료됨

## 🚨 재작업 트리거

다음 조건 중 하나라도 해당되면 이전 작업물을 재작업해야 함:

- [ ] 수용 기준이 구현되지 않음
- [ ] 단위 테스트가 실패함
- [ ] 통합 테스트가 실패함
- [ ] 아키텍처 가이드라인을 위반함
- [ ] 발생한 이슈가 해결되지 않음
- [ ] 다음 에이전트가 검증할 정보가 부족함

---

## 📦 커밋 및 버전 관리

- **브랜치**: `feature/STORY-[번호]`
- **커밋 시점**: TDD 사이클에 따라 기능 구현과 테스트 작성이 완료되고 자체 품질 게이트를 통과했을 때.
- **커밋 메시지**: `Dev: [구현 기능 요약] 기능 구현 완료 (#STORY-[번호])`
- **설명**: 특정 기능의 구현 완료를 기록합니다. 작은 단위로 커밋하여 코드 리뷰와 롤백을 용이하게 만듭니다. (예: `Dev: 반복 일정 생성 로직 구현 완료 (#STORY-123)`)
